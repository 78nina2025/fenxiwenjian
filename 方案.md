## 1. 目标与范围

### 1.1 目标
从 62 本中文量价/价量分析类股票书中系统化抽取全部可判别的**多空判断规则**,形成结构化、可检索、可回测的规则库,并以此为核心产物编写系统规格说明书。

### 1.2 成功判据
1. 规则库字段完整(见 §6 schema),每条规则可溯源到具体书页。
2. 规则之间可检测冲突与重复(合并去重报告可生成)。
3. 至少 30% 的规则可以被直接翻译成 pandas/vectorbt 向量化表达式并跑通最小回测。
4. SRS 文档通过"自查清单"(IEEE 29148-2018 量表,§9)。

### 1.3 非目标(本期明确不做)
- 不做实盘交易系统的接单/风控/清算层。
- 不做规则的有效性/alpha 评估(仅提供评估**入口**与字段,不出"哪些规则赚钱"的结论,以免错用 in-sample 结果)。
- 不做规则作者观点的对错判断,仅记录"书里怎么说"。

见 `book_list.txt`(同目录),62 个文件。页数分布摘录:

特征说明:
- 文件名普遍带"(高清)"/"(高彩)",实为扫描件。

## 4. 技术选型与理由

### 4.1 OCR 路线对比

| 方案 | 精度(中文) | 速度(M系列) | 成本 | 隐私 | 决策 |
|---|---|---|---|---|---|
| **PaddleOCR(PP-OCRv4)** | 95~97%(印刷中文) | ~2~4 页/秒(CPU)、~8~12 页/秒(Metal/MPS) | 0 | 本地 | **首选** |
| Tesseract 5 + chi_sim | 88~92% | ~0.5~1 页/秒 | 0 | 本地 | 备用 |
| 百度高精度 OCR API | 98~99% | 网络受限,~5 页/秒 | ≈0.01 元/次 ≈¥163 总 | 上云 | 预算充足时优选,单本内快速验证 |
| 阿里/腾讯 OCR | 97~99% | 类似 | 类似 | 上云 | 备选 |

**最终选型**:PaddleOCR 作为主链路;保留百度 OCR 作为失败页重试兜底。理由:
- 书页质量不均,PaddleOCR 版面分析(layout parser)能更好处理图表夹杂的页面。
- 无需数据出境,62 本 × 几百页如果上云会涉及版权争议。

### 4.2 规则抽取:为什么必须用 LLM

传统 NER / 正则只能匹配明文术语,书中规则大多是**多句论述型**,例如:
> "当股价在经过较长时间的横盘震荡后,某日成交量突破前 20 日均量的 2 倍以上,收盘收在前期高点之上,且第二日能够高开不回补前日阳线一半者,可视为有效突破。"

必须用具备长程理解能力的 LLM 按结构化 schema 抽取。选型:**Claude Sonnet 4.6**(claude-sonnet-4-6)。

- 使用 **prompt caching**(书级上下文做 cache,每本书内多段抽取时缓存命中 ≥90%,成本下降约 8~10 倍)。
- 每次抽取输入 ≈ 1 本书单章 OCR 文本(8~15K tokens),输出 JSON 规则数组。
- 按书分片 + 重叠 500 字符,避免跨章节漏抽。

### 4.3 指标算子与回测库

依据调研(见 `文献与开源库.md` §B):

- **指标算子底座**:TA-Lib(C 核,150+ 指标)+ stockstats(字符串 DSL)+ pandas-ta(补齐)。
- **回测**:vectorbt(向量化参数扫描)+ nautilus-trader(事件驱动,供未来接实盘)。
- **规则因子化评估**:alphalens-reloaded。
- **数据源**:akshare(主)+ tushare Pro(备)+ efinance(盘口)。
- **规则 DSL**:优先 pandas query;复杂前向链时引 durable_rules。

### 4.4 不选的理由

- **不选纯 Python OCR(如 easyocr)**:中文印刷书场景实测精度 3~5 个百分点落后 PaddleOCR。
- **不选 backtrader 作主回测**:GPL-3 传染,且主仓停滞。
- **不选 zipline 原仓**:已停止更新,改用 zipline-reloaded。
- **不选重型规则引擎(Drools/CLIPS)**:规则数量 <2k,pandas query 足够,避免运维成本。

## 5. Pipeline 分阶段设计

### 5.1 Phase 1 — PDF 拆页 + OCR(瓶颈阶段)

脚本:`pipeline/01_ocr.py`

- 并发策略:进程池大小 = CPU 核数 - 2,避免 OOM(PaddleOCR 单进程 1~2 GB RAM)。
- 每页输出:`ocr_out/<book_id>/page_<n>.txt` + `page_<n>.json`(含 bbox)。
- 水印过滤:按固定字符串(已识别的 "股票开户""QQ/微信:958218088" 等)正则剔除。
- 进度:每书一个 `progress.json`,支持断点续跑。

**估时**:M2 Pro 10 核,CPU 混合精度 PP-OCRv4 ≈ 3 页/秒 → 16,260 / 3 ≈ 5420 秒/进程 → 8 并发约 **12~15 小时**(实际受 I/O、热节流影响,放宽到 20h)。若改 GPU/MPS 预计 6~8 小时。

### 5.2 Phase 2 — 清洗 + 分段

脚本:`pipeline/02_clean.py`

- 全角/半角归一化、去页眉页脚、去页码、合并断字段落。
- 章节/小节边界识别(规则:行首数字编号 + 短标题 + 换页)。
- 输出:每本一个 `clean/<book_id>.jsonl`,每行一个 section,字段 `{book_id, section_id, page_start, page_end, title, text}`。

**估时**:纯 CPU pandas 处理,≤ 1 小时。

### 5.3 Phase 3 — LLM 规则抽取

脚本:`pipeline/03_extract_rules.py`

- 输入:每本书的 sections(按 12K token 窗口拼接)。
- System prompt(做 cache):本规则库的 schema + 抽取示例 + 术语表。
- User prompt:一段书文本 + "抽取多空判断规则,按 JSON schema 返回"。
- Schema 见 §6;必填字段、枚举字段严格校验。
- 失败重试 3 次,指数退避;失败条目进入 `errors/`。

**估时与成本**(Claude Sonnet 4.6,按当前定价估算):
- 输入总量 ≈ 16,260 页 × 600 tokens ≈ **9.8M tokens**
- 输出 ≈ 每本约 1500 tokens × 62 ≈ **0.1M tokens**
- 缓存命中假设 85%:未缓存 ≈ 1.5M × $3/MTok = **$4.5**,缓存读取 ≈ 8.3M × $0.30/MTok = **$2.5**,输出 0.1M × $15/MTok = **$1.5**
- 合计 **≈ $8.5(¥60)**。若走 Opus 4.7 则约 **¥350~600**。
- 挂钟时间:单线程 ~8~12 小时,10 并发 ~1~2 小时。

### 5.4 Phase 4 — 合并去重

脚本:`pipeline/04_merge.py`

- 为每条规则计算句向量(推荐 `bge-small-zh` 或调用 Claude `embedding` 端点)。
- 同主题组内用 HDBSCAN 聚类;同簇内保留"最长、出处最多"的条目为主,其余归为 `variants`(副本)。
- 冲突检测:同前置条件下方向相反的规则进入 `conflicts.csv` 人工裁决。

**估时**:本地 2~3 小时。

### 5.5 Phase 5 — 导出

脚本:`pipeline/05_export.py`

- 用 openpyxl 写 `rules.xlsx`(多 sheet:规则主表、变体、冲突、来源索引、术语表)。
- 用 python-docx 生成 `rules.docx`(按章节分类的正文式规则书)。
- 生成 SRS 文档的"规则库快照"附录章节。

### 6.2 分类体系(主 → 子)


## 8. 合并去重与冲突处理

### 8.1 去重
- 句向量 cosine >0.88 且类别/方向一致 → 判定同类,保留最详细者。
- 合并时保留所有来源(一条规则可能有 30+ 出处)。

### 8.2 冲突
典型冲突:"放量上涨视为多头"vs"放量冲高但收十字星视为顶部"。处理:
- 同前置条件下方向相反的规则进入 `conflicts.csv`。
- 人工裁决后添加 `tie_breaker_condition` 字段区分适用场景。

### 8.3 覆盖度评估
- 按 §6.2 分类表统计每个子类的规则数。
- 若某子类 <5 条 → 提示 LLM 二次抽取。

## 11. 风险登记

| # | 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|---|
| R1 | OCR 扫描质量差,错字率 >5% | 中 | 高 | 失败页走百度高精度 OCR 兜底 |
| R2 | LLM 抽取幻觉字段 | 中 | 中 | schema 严校验 + 出处页码强制回指 + 句向量比对原文 |
| R3 | 规则方向性判断错误 | 中 | 高 | 人工审 10% 抽样;direction/invalidation 双字段交叉校验 |
| R4 | 中文简繁/异体字不统一 | 低 | 低 | 统一转简体 + OpenCC |
